Amazon Price Checker

Team members: Tim Cheung, David Dang, David Wong
Duration of project: May 14-15, 2016

Introduction - This website was developed for the Angular Attack 48 hour online Angular Hackathon. This website checks whether buying from Amazon.ca is cheaper than Amazon.com.

Description - Compare the product prices on Amazon.com and Amazon.ca to see if it is cheaper with the current exchange rates. (Safari is not supported)

Instructions - Search for a product you wish to buy on Amazon.

Built with

  • Digital Ocean server hosting a Node JS amazon product api client, running on Koa.
  • This starter pack, which includes Angular 2, SystemJS, Gulp, and Bootstrap

Robotic Boat

Team members: Jaclyn Tarnai, Graham Williamson, David Wong, Yiran Zhou
Duration of project: September 2015 - April 2016

Abstract - The goal of this project is to create a low cost autonomous marine vessel for surveying the bottom of lakes with a bathymetry sensor. The main objectives for this project are autonomous GPS waypoint navigation for a cost less than $2,750. The final design was a 2m long catamaran vessel, with a top speed of 8km/h.

Introduction - The client, Stratus Aeronautics, was considering the development of a low cost autonomous marine vessel for operation on lakes within Canada. Existing bathymetry products were prohibitively expensive, or lacked desired functionality. Therefore, development of a low cost custom designed vessel was required. To complete this project, the design team had to meet the following objectives:

  • Device operates autonomously
  • Device costs less than $2,750 CAD
  • Device is in the order of 1.8m x 0.9m
  • Device can be transported in the rear of a truck
  • Device travels greater than 5 km/h
Following completion Stratus will use the boat to perform bathymetry surveys of freshwater locations, as a part of their unmanned surveying business.

Design overview - The team designed and constructed a 2m long catamaran vessel, which is capable of holding equipment adding up to a total max weight of 45.6kg. The vessel has two rudders and two thrusters placed between the hulls. Five LiFePO4 batteries are used to power the electronics. The LiFePO4 batteries are placed inside the hulls for stability. Two Pelican watertight cases are used to house the electronics. Two 4-cell LiPO batteries are used separately to power the payload. The payload is an M3 multi-beam echosounder provided by Kongsberg Mesotech. The boat is controlled with a Pixhawk Autopilot by setting GPS waypoints.

Technical analysis and testing - An analytical model of the boat was created to optimize the boat dimensions for best performance and to identify the most critical and risky criterion. An example of the graphical output from the analytical model is provided below. Five tests were conducted to validate and verify that the boat meets the needs and requirements set:
  • Critical function prototype on the autonomous navigation system
  • Scale model drag test
  • Final prototype float test
  • Final prototype trial test at Alta Lake
  • Final prototype trial test at Como Lake
Results:
  • Maximum speed of approximately 2.2 m/s (8 km/h)
  • Turning radius of approximately 1.7 m
  • Operating time of about 2 hours (for sensor)
  • Calculated operating time 10.66hr with lithium batteries
Discussion and conclusion - The measured boat weight was initially predicted to be reduced by close to 10kg by switching to lithium batteries from lead-acid. However more durable features were added, which cut into expected weight savings. The main functions of the boat are currently in operation, however some tuning is still required for the autopilot system. The design team have not been able to confirm the actual boats battery life since the capacity is so large.

Sunlight Position Simulator

Team members: Zhenhe Lu, David Wong
Duration of project: March 2016 - April 2016

Abstract - The objective of this project is to physically simulate the position of the sunlight relative to the Earth. Typically, in solar system models, the Earth revolves around the sun and the Earth rotates at an angle of approximately 23.5 degrees. Therefore, this can be difficult for people to understand why the amount of daylight varies depending on the time of the year, and also how and why time zones exists.

For this model, the reference frame is changed from the sun to the Earth’s axis of rotation to minimize confusion. The model will also exaggerate the sunlight concentration to highlight how the sunlight moves with time. To demonstrate the concept of time zones and how the amount of daylight varies, the following features was implemented:

  • Automatic rotation of the Earth and automatic light position adjustment
    • This feature simulates the Earth’s rotation and how the sunlight position varies with time
  • Location lookup and date lookup
    • This feature allows the user to lookup a location to obtain the sunrise, sunset, twilight begin, and twilight end of a certain location at a certain date. As well, the Earth with rotate to that location and display where the sunlight position will be.
Objective - The objective of this project is to build a sunlight position simulator to demonstrate the concept of time zones and how the amount of daylight varies. The sunlight position similar uses a desktop globe that rotates either automatically or according to the date and location from user input on the software. A stepper motor is used control globe’s angular position. As well, an array of LEDs is used to simulate the sunlight location relative to the earth. A user interface software was designed and used read user input and send signal to microcontroller to rotate the globe and light up LEDs in order to represent a miniature of earth-solar system.

Rationale - This project physically simulates the position of the sunlight relative to the earth, so it is useful for education purposes such as astronomy. The user can understand the concepts of time zones and seasons due to the physical representation. Lastly, this project makes for a great art piece compared to a standard clock or calendar. Compared to other models, this project changes the reference frame from the sun to the Earth’s rotation axis to minimize confusion. As well, this project uses a physical model controlled by software to make it easy to visually understand the concept of time zones and seasons.

Functional Requirements - The following functional requirements were met in order to complete the project:
  • Programming the MSP430 microcontroller to control the stepper motor
  • Designing the LED array to simulate the sunlight
  • Developing a C# program for the user interface and system integration
  • Designing and fabricating the mechanical components and assembly
  • Developing the code to determine the Earth's rotation and sunlight location

Clefairy Says using an Accelerometer

Team members: David Wong
Duration of project: January 2016

Introduction - This program uses a 3-axis accelerometer as a controller to play the Nintendo 64 Pokémon Stadium mini-game, Clefairy Says. The program was developed in C# using Visual Studios.

Design overview - The C# program reads the data from the accelerometer in the following sequence: Start byte, X-acceleration byte, Y-acceleration byte, Z-acceleration byte. The following features are implemented for handling the acceleration data:

  • Displaying current value and standard deviation for each axis
  • Displaying current orientation
  • Displaying the last 100 values for each axis on a line graph
  • Saving the values to a CSV file
  • Recognizing acceleration sequences as gestures and displaying the gesture
From this, the Clefairy Says game can utilize the gesture recognition as the Up, Down, Left, Right inputs. The C# program will start the game with the sound, graphics, music, and animations similar to the actual Nintendo 64 game as shown in the following YouTube video: https://www.youtube.com/watch?v=9hNphbAm_Hs

UV Light Fixture

Team members: David Wong
Duration of project: May 2015 - August 2015

Abstract - The purpose of the UV light fixture is to demonstrate a photolithography process on a three dimensional substrate, such as a cylinder, to fabricate devices, such as stents. The characteristic and performance of the UV light fixture is determined by conducting various exposure tests on various photoresist. This project was done at Takahata Lab at UBC.

Design overview - The UV light fixture is intended to work with the micro EDM at Takahata Lab. The circuit was designed to allow the following features:

  • Manually turning on or off the UV light
  • EDM controlling the on or off state of the UV light
  • Electrically decoupling the EDM and UV light
  • Allowing the EDM or user to control UV light
  • Indicator LED to determine if UV light is on
The fixture was mechanically designed to allow the following features:
  • Rigidly mounting onto the microscope stage on the EDM without interfering the microscope
  • Rigidly mounting the mask using a mask holder
  • Low cost
Operation - The fixture operates using two switches: an on-off switch, and a toggle switch. The toggle switch allows the UV light fixture to be operated on and off by the micro EDM or by the user. The on-off switch turns the UV light on or off. An indicator LED safely shows whether the UV light is on or off. It is recommended to wear UV light protection during operation.
